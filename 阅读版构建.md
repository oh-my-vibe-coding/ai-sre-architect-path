# 阅读版构建

> [← 返回目录](README.md)

本文档说明：**如何从 Markdown 源生成更漂亮的在线阅读站点和 PDF**。

> [!IMPORTANT]
> **Markdown 是唯一源文件**。`mkdocs.yml` / `build/*` / `site/` / `output/*.pdf` 都是**派生产物**，书稿语义只在 `.md` 文件里。

---

## 为什么保留 Markdown 作为源

- **可 diff / 可 review**：任何人都能看到 PR 变了什么字
- **不依赖渲染器**：GitHub / Obsidian / Typora / VS Code 都能开
- **长期稳定**：工具可能消失，文本格式不会
- **在线站点和 PDF 都是阅读层**：好看但不 authoritative

---

## 在线阅读站点 · MkDocs Material

### 依赖

```bash
python -m venv .venv
source .venv/bin/activate          # Windows 用: .venv\Scripts\activate
pip install mkdocs mkdocs-material pymdown-extensions
```

**可选**（增强渲染）：

```bash
pip install mkdocs-material[imaging]   # 社交卡 / 图像优化（可选）
pip install pillow cairosvg            # 图像相关依赖
```

### 本地预览

在书根目录：

```bash
mkdocs serve
# 打开 http://127.0.0.1:8000
```

改 `.md`、`extra.css`、`hooks/*.py` 任何文件会**热刷新**。

### 构建静态站

```bash
mkdocs build
# 输出到 site/
```

产物是纯静态 HTML + JS + CSS，可直接：
- `python -m http.server -d site 8080` 本地看
- rsync 到任何静态托管（GitHub Pages / Cloudflare Pages / Netlify / nginx）

### 一键脚本

```bash
bash build/build-site.sh
```

## PDF · Pandoc + XeLaTeX 路线

### 依赖

**pandoc**：
```bash
brew install pandoc            # macOS
sudo apt install pandoc        # Ubuntu
```

**XeLaTeX**（需要较大磁盘，~4 GB）：
```bash
brew install --cask mactex-no-gui   # macOS（推荐 no-gui，省 3 GB）
sudo apt install texlive-xetex texlive-fonts-recommended texlive-lang-chinese   # Ubuntu
```

**字体**（PDF 渲染中文必需其一）：
- **Noto Serif CJK SC** / **Noto Sans CJK SC** — Google Fonts 免费
- **Source Han Serif SC** / **Source Han Sans SC** — Adobe 开源
- **Songti SC** / **PingFang SC** — macOS 自带

macOS 装 Noto CJK：
```bash
brew install --cask font-noto-serif-cjk-sc
brew install --cask font-noto-sans-cjk-sc
```

Ubuntu：
```bash
sudo apt install fonts-noto-cjk fonts-noto-cjk-extra
```

### 构建主书版 PDF

```bash
bash build/build-pdf.sh
# 输出到 output/AI时代SRE架构师之路-v1.8.5.pdf
```

**默认包含**（主书版，**不是**全量）：
- 前言 + 引章
- 第一部分 · 处境与角色
- 第二部分 · 核心能力
- 第三部分 · 架构决策
- 第四部分 · 训练与落地（含 Unit 总览、贯穿项目、Capstone）
- 附录 A（月度自检）/ E（模板库）

**不含**（有需要再做参考卷）：
- 深入 01-19、科学 01-04、共同语言 01-05（专题卷，可各自单做 PDF）
- 代码 GUIDE（IDE 里看体验更好）
- 复习 / 维护（日常工具属性，不是长阅读）

**要全量 PDF**：改 `build/build-pdf.sh` 里 `FILES` 数组或加参数。

### 为什么这么拆

80+ 个 Markdown 全塞进一个 PDF：500+ 页、无人读完、生成慢、PDF 反而不便查询。主书版聚焦**四部分主线**；专题卷按需生成。

## 备选 PDF 路线

`build/build-pdf.sh` 主路径是 **Pandoc + XeLaTeX**。如果你的环境装不了 TeX：

### 路线 B · Chrome Headless（无 LaTeX 依赖）
- 利用 MkDocs 生成的 HTML，用 Chrome `--print-to-pdf`
- 质量不如 LaTeX，但无 TeX 环境可用
- 本项目的早期 PDF 用了此路线（见 `build/README.md`）

### 路线 C · Typst（新兴，更快）
- [Typst](https://typst.app/) 有原生 CJK 支持，编译比 LaTeX 快 10×
- 需要把 Markdown 转 Typst，学习成本中等
- 适合未来想要更漂亮排版时切换

### 路线 D · Vivliostyle（CSS 排版引擎）
- 用 CSS 控制分页 / 页眉 / 页脚 / 双栏
- 适合已有 CSS 功底的维护者

**当前建议**：**Pandoc + XeLaTeX 稳定优先**；想换 Typst 再说。

## 常见问题

### Q1 · 中文文件名在 URL 里会不会乱码
不会。现代 MkDocs 对 Unicode URL 友好。但如果部署到严格静态主机，可能需要 URL-encode。**推荐通过 MkDocs 的站内搜索而非外链访问具体页面**。

### Q2 · Mermaid 图不渲染
检查 `mkdocs.yml` 里 `markdown_extensions` 是否有：
```yaml
- pymdownx.superfences:
    custom_fences:
      - name: mermaid
        class: mermaid
        format: !!python/name:pymdownx.superfences.fence_code_format
```
本项目**已配置**。如果仍不渲染，`mkdocs serve` 控制台看报错。

### Q3 · 表格太宽溢出
本站点 CSS 已把 `.md-typeset__table` 设为可横向滚动。如果仍破版，考虑在源 MD 里：
- 减少列数
- 用 `<br>` 在单元格内换行
- 拆成多张表

### Q4 · GitHub Alerts 不渲染成 callout
源 Markdown 用 `> [!NOTE]` 这种语法 —— MkDocs 原生不识别。本项目通过 `hooks/github_callouts.py` **预处理**转成 MkDocs admonition。如果你加了新 alert 类型（如 GitHub 后来增的），改这个 hook 即可。

### Q5 · PDF 生成报 `Font not found`
XeLaTeX 找不到字体。检查：
- `fc-list :lang=zh | grep -i noto` 能否找到 Noto CJK
- 如果没有，按上方安装 `font-noto-serif-cjk-sc`
- 或改 `build/pdf-style.tex` 里的 `\setCJKmainfont` 为你系统已有的字体

### Q6 · 我想改书内容怎么办
**改 `.md` 源文件**。不要改 `site/` 或 `output/*.pdf`——那是派生物，下次 build 会被覆盖。

## 发布工作流建议

```
改 .md              —— 唯一源
    ↓
mkdocs serve        —— 本地预览
    ↓
mkdocs build        —— 输出 site/
    ↓
推送到静态托管      —— GitHub Pages / CF Pages
    ↓
bash build/build-pdf.sh  —— 生成 output/*.pdf
    ↓
发 release          —— 附 PDF 作为 artifact
```
