# 前言 · 为什么需要这本小书

> [← 返回目录](README.md)

这本书不是"AI 工程百科"，也不是"如何使用大模型"的指南。它服务一个具体的人：一位 SRE，日常重度依赖大模型写代码和操作线上环境，并且已经察觉到自己正在**失去专注力和独立学习能力**。

如果你正是这样的人，这本书试图回答**三个**问题：

1. **在 AI 已经承担大量日常操作的时代，一名 SRE 架构师应该具备什么能力？**
2. **如何在重度使用 AI 的同时，依然保留并磨砺自己的思考能力？**
3. **如何与算法工程师 / 科学家建立共同语言，从"伺候 ML"的角色升级为"和 ML 并肩架构系统"的伙伴？**

这三个问题对应本书的三条主线：

- **Track A · 能力**：建立 AI 时代 SRE 架构师的工程能力体系（知识 / 深入 / 科学）
- **Track B · 学习**：在重度使用 AI 的同时磨砺思考能力（理念 / 练习）
- **Track C · 协作**：掌握与 ML 工程师对话的共同语言（共同语言）

三条主线**不是并列的三套作业**，而是**彼此咬合**：
- Track B 是 Track A 的执行方式（自己写、AI 挑错）
- Track C 建立在 Track A 的工程基础上（没有架构感就没法和 ML 真协作）

## 如何阅读

- **Track A · 能力**：
  - **理念（第 1-3 章）**—— 态度校准，15 分钟读完
  - **知识（第 4-9 章）**—— 能力画像六维，按需查阅
  - **深入（12 篇）**—— 工程级专题（TTFT、Caching、容量、Eval、红队、记忆、事故模式……）
  - **科学（4 篇）**—— 恰到好处的机制（Attention、长 context、量化、tokenization）
- **Track B · 学习**：
  - **练习（第 10 章 + 6 Units + Capstone）**—— 每周怎么动的执行手册
- **Track C · 协作**：
  - **共同语言（5 篇）**—— 训练、数据、研究级 eval、Alignment、分布式训练

**不要一口气读完**。这本书是用来"陪你走 4-5 个月"的，不是用来读完的。
