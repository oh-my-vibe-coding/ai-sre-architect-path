## 如何使用这本书

1. **第一次打开**：读前言 → 引章 → [学习路线图](学习路线图.md) → 第 1-3 章（理念部分）。大约 20 分钟。
2. **开始学习**：直接进 [练习/周循环总览](练习/周循环总览.md)，跟着"第一周任务"开跑。不要先读完所有深入专题。
3. **最大化学习效果**：用 [🛠️ 贯穿项目 · SRE 事故助手](练习/贯穿项目-SRE事故助手.md) **陪跑每个 Unit**——每周 B2 产出落到项目里，比只写文档学到的多一个量级。
4. **每周一次**：打开当前 Unit 的当前 Week 文件，照着做。**不用提前看后面几周**。
5. **知识章节（第 4-9 章）按需读**：学到哪个 Unit，对应回查哪一章。不求通读；对应关系见 [学习路线图](学习路线图.md)。
6. **月末**：打开 [每月自检表](附录/A-每月自检表.md)。
7. **终点**：完成所有 Unit 后，做 [🏆 Capstone · 架构评审包](练习/Capstone-AI生产架构评审包.md)（首选对象 = 贯穿项目）。

## 不同读者怎么走

本书的主体学习路线是 **19 周**（Unit 0-5），之后用 **Capstone** 做一次综合评审。不是所有读者都需要走全。

| 读者类型 | 核心路线 | 重点关注 | 可跳过 |
|---|---|---|---|
| **传统 SRE**（想进 AI 时代） | 完整 19 周 + Capstone | 三条主线全走 | 无（都有价值）|
| **平台工程师**（做 LLM 平台 / 网关） | Unit 0 + **Unit 2 / 3 / 5** | 推理 SLO、容量、静默降级、Eval 流水 | Unit 4（应用级），共同语言 01/04 可浅读 |
| **AI 应用工程师**（做 Agent / RAG） | Unit 0 + **Unit 1 / 2 / 4** | Agent 自治、Eval、复合可靠性 | Unit 5（数值级），共同语言 05 |
| **技术管理者 / 架构师** | 理念 + 知识 + **Mastery Gates** + **Capstone** | 整体框架、决策判据、事故模式 | 具体代码实现、Unit 周任务可快读 |
| **算法 / ML 工程师**（理解 SRE 视角） | **共同语言** 5 篇 + Unit 2 / 3 / 4 | SRE 怎么看 ML 系统、Eval SLO | Unit 0（已会）、Unit 5（你已是专家）|

**选路建议**：
- **不确定自己属于哪类？** 走传统 SRE 完整路线，不会亏
- **想最短时间建立判断力？** 读理念 + 所有 Mastery Gate + Capstone，约 1 周
- **想针对当下工作？** 按项目需求挑 1-2 个 Unit 深做
- **自学者（没有团队 / 公司项目陪跑）？** **强烈建议选 [🛠️ 贯穿项目](练习/贯穿项目-SRE事故助手.md) 作为主线**，否则练习容易停留在纸面

**无论走哪条**：[复习系统](复习/README.md) 都值得用；[附录 E · 模板库](附录/E-模板库.md) 都能直接拿走。

## 本书不教什么

明确预期，免得你期待错位：

- ❌ **从零训练大模型** — 不是 ML 教材。想训练看 HuggingFace LLM Course / Andrej Karpathy 的视频
- ❌ **Prompt 技巧大全** — 只讲 SRE 视角用到的基础 prompt。深入看 Anthropic / OpenAI 的 prompt engineering 课程
- ❌ **每日模型榜单追踪** — [深入 03](深入/03-模型与工具场景化最佳实践.md) 是月度更新的快照，不是每日播报
- ❌ **完整 MLOps 平台建设** — 只讲 SRE 视角的 LLM 部分。想建 MLOps 平台看 MLOps Community 的材料
- ❌ **替代 Google SRE Book** — 经典 SRE 原则（SLO / IC / Postmortem）本书只做**AI 化扩展**，不重复基础
- ❌ **ML Research 能力训练** — 共同语言部分让你**听懂**研究，不让你**做**研究
- ❌ **通用编程 / 分布式系统入门** — 默认你已有工程基础

**本书的核心野心**：让一位已经是 SRE 的工程师，在 AI 时代**不被替代、能与 ML 同事并肩、能做出架构判断**。比这大的目标，请看别的书。
