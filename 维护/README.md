# 维护系统 · README

> [← 返回目录](../README.md)

> [!IMPORTANT]
> **AI 技术迭代极快**。没有维护机制的书**写完即过期**。

## 基本节奏

```
每月第一个周一
   ↓
月度更新清单（15 min）──► 更新快照 → 更新 CHANGELOG → bump 版本 patch

每季度第一个月
   ↓
季度审查清单（2 hr）  ──► 深度复查 → 可能更新框架 → bump 版本 minor

每年一次
   ↓
整本书框架重审（1 day）→ bump 版本 major
```

---

## 版本号约定

按 **Semantic Versioning** 简化使用：

- **Major** (v1 → v2)：框架重构、主线变化、整章删改
- **Minor** (v1.1 → v1.2)：新增深入 / 科学 / 共同语言章节
- **Patch** (v1.1.0 → v1.1.1)：快照数据更新、模型 / 工具更新、勘误

**当前版本**：`v1.3.2` · 2026-05-24

> 版本同步原则：改了 README 的版本号，**必须**同步更新此行。否则读者困惑。

## 漂移度分类

本书内容按更新频率分四级（详见 [漂移度表](漂移度表.md)）：

| 级别 | 标记 | 含义 | 举例 |
|---|---|---|---|
| 🧱 **基石** | stable | 几乎不变（>1 年） | 理念、Transformer attention、Tokenization BPE |
| 🔩 **缓变** | slow | 季度级变（3-6 月） | 场景最佳实践、Eval pipeline 设计 |
| 📊 **快变** | volatile | 月度级变（1-2 月） | 模型排名、定价、工具版本 |
| 🕒 **极不稳** | snapshot | 周级别变（数周） | LM Arena 分数、SWE-bench Top 5 |

**读者怎么用**：
- 看到 🧱：放心学
- 看到 🔩：关注但不急
- 看到 📊：注意日期，超过 2 月最好查原链接
- 看到 🕒：**一律查原链接**，本书数字仅作参考

## 如果是你自己（个人维护者）

建议把维护当成 [Track B 学习法](../练习/10-三个核心训练动作.md) 的一部分：
- **月度更新 = B3 深度块**（无 AI 读一手材料的机会）
- **季度审查 = B2 产出**（写更新说明就是写作训练）
- **年度重审 = 整本书的 mastery gate**

**维护的过程就是学习的过程**。别外包给 AI 自动跑——就跟 B2 一样，让 AI 挑错可以，让它独自更新不行。

---

## 如果是团队（多人协作维护）

- 每个章节有 **owner**（frontmatter 加 `maintainer` 字段）
- **Pull request 模板**放 [更新模板](更新模板.md)
- CHANGELOG 每次更新必须同步 bump
- 重大变更（minor / major）必须过 **2 人 review**
