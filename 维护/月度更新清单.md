# 月度更新清单

> [← 维护系统](README.md)

> [!NOTE]
> **每月第一个周一，花 15-20 分钟过一遍**。
>
> 不做这个 → 两三个月后书就过期、读者流失。

---

## 预期时间：15-20 分钟

- 查 5 个网站：~5 分钟
- 更新 2-3 个章节快照：~10 分钟
- 更新 CHANGELOG + 版本号：~2 分钟
- Commit：~1 分钟

---

## Part A · 快照数据更新（每月必做）

### A1 · 模型榜单（深入 03）

- [ ] 打开 https://arena.ai/leaderboard
- [ ] 记录当前 Overall Top 10 的前 3 名（只要前 3 变了就更新）
- [ ] 记录 Coding Top 5
- [ ] 如有**新模型进榜单**或**分差 >30**，更新 深入 03 · §3.11

- [ ] 打开 https://openrouter.ai/rankings
- [ ] 过滤付费模型，看 Top 10 变化
- [ ] 更新 深入 03 · §1.5 的表格

- [ ] 打开 https://www.swebench.com/verified.html
- [ ] 看 Verified Top 5 有无变化
- [ ] 更新 深入 03 · §3.11（SWE-bench 表）

### A2 · 厂商定价 / 新模型（深入 03）

对每家厂商 docs 查一眼（1 分钟/家）：
- [ ] Anthropic: https://platform.claude.com/docs/en/docs/about-claude/models
- [ ] OpenAI: https://platform.openai.com/docs/models
- [ ] Google: https://ai.google.dev/gemini-api/docs/models
- [ ] DeepSeek: https://api-docs.deepseek.com/
- [ ] 中国系（Moonshot / MiniMax / 智谱 / 腾讯）跑个 WebSearch 看有无大版本

**发现新模型** → 加进 深入 03 · §3 相应厂商区

### A2b · 三大模型系列口径（深入 12）

- [ ] Claude：Opus / Sonnet / Haiku 命名、context、extended thinking、tool use、prompt caching 是否有变化
- [ ] OpenAI：GPT / mini / nano / reasoning / Codex 命名、Responses API、structured outputs、reasoning effort 是否有变化
- [ ] Gemini：Pro / Flash / Flash-Lite / Live 命名、long context、thinking、function calling 是否有变化
- [ ] 如有变化，同步更新 [深入 12](../深入/12-Claude-GPT-Gemini三大模型系列使用指南.md)、[附录 C](../附录/C-术语表.md)、[附录 D](../附录/D-厂商官方学习资源.md)

### A3 · 工具生态（深入 03）

- [ ] `gh repo view anthropics/claude-code` 查 star 数 + 最新 release
- [ ] 同样对 openai/codex, google-gemini/gemini-cli, sst/opencode, cline/cline
- [ ] 有重大新工具出现 → 加进表格

### A4 · 附录 B 链接存活性

- [ ] 抽样 10 个 参考文献 的链接是否仍然有效（改天写个脚本自动扫）
- [ ] 失效的链接 → 在附录 B 标注 `[broken]` 或找替代

### A5 · 复习系统校对

- [ ] 打开 [核心概念卡](../复习/核心概念卡.md) 与 [anki-import.csv](../复习/anki-import.csv)
- [ ] 检查涉及 **模型命名、价格、context、厂商分层** 的卡片是否过时（Gemini Pro/Flash 命名、Claude Opus/Sonnet 当前旗舰、GPT 价格、tokenizer 代际差异）
- [ ] 涉及 **快变事实** 的卡片要么更新、要么改写成"问机制不问数字"
- [ ] **核心概念卡和 csv 必须同步**：在概念卡里改了任何 Q/A，对应的 csv 行也要同步。否则 Anki 用户的卡片和 markdown 不一致

## Part C · 收尾（每月必做）

### C1 · 更新 CHANGELOG

在 [CHANGELOG.md](CHANGELOG.md) 里加一段（复制 [更新模板](更新模板.md)）：

```markdown
## v1.0.X — YYYY-MM-DD

### 更新
- 深入 03 · 模型榜单月更（LM Arena top 1 换为 X）
- 深入 03 · Anthropic Haiku 4.6 发布，加入表格
- 附录 B · 2 个失效链接替换

### 发现
- [记录"这个月有什么值得关注但还没进书"的事]
```

### C2 · Bump 版本号

- [ ] README.md 里的版本号：v1.0.X → v1.0.(X+1)
- [ ] CHANGELOG 里的对应条目

### C3 · 同步 frontmatter

- [ ] 被修改的章节 `updated:` 字段改成今天
- [ ] README.md 的 `updated:`

### C4 · Commit（如果用 git）

```bash
git add -A
git commit -m "chore: monthly update YYYY-MM (v1.0.X)"
# 详细内容在 CHANGELOG 里
```

## Part D · 如果这次发现了"不止月度级"的变化

有时候月度检查会发现**范式级变化**（比如"整个榜单洗牌了"或"新架构横空出世"）。

**不要强塞进 patch**。标记一下，下次**季度审查**时系统性消化：

- [ ] 在 [CHANGELOG · Unreleased 段](CHANGELOG.md#未发布unreleased) 加一条"待季度消化"
- [ ] 或建一个 `维护/待办.md` 收集

## 工具辅助（推荐）

### 自动化脚本（不必须，但值得写一次）

你可以写一个 bash + curl 脚本做 A1 的前 3 项：

```bash
#!/bin/bash
# check-month.sh —— 月度检查辅助

echo "=== LM Arena (打开浏览器看) ==="
open https://arena.ai/leaderboard

echo "=== OpenRouter ==="
curl -s https://openrouter.ai/rankings | head -50

echo "=== SWE-bench ==="
open https://www.swebench.com/verified.html

echo "=== 附录 B 链接扫描 ==="
awk -F'[()]' '/\(http/ {print $2}' 附录/B-参考文献.md | while read url; do
  curl -s -o /dev/null -w "%{http_code} $url\n" "$url"
done
```

## 节奏失控时

**3 个月没做月度更新**？别堆着。做一次**季度审查**（见 [季度审查清单](季度审查清单.md)）把积累一起消化。

**6 个月没做**？考虑整体重审。发一个 minor 版本（v1.0 → v1.1）。

**1 年没做**？书的可信度已经严重受损——这时候要么做 major 重构，要么标注 `DEPRECATED`。
