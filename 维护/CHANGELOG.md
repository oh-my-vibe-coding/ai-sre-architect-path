- **为什么是 minor 而非 major**：内容上是"7 篇新章节 + 4 篇骨架填实"——这符合 minor = "新增章节"。整书结构（理念/知识/架构/练习/深入/科学/共同语言/复习/代码/附录/维护）没有改动，所以不是 major。
- **架构 01-04 vs 架构 05-07 的分工**：01-04 是"现状 + 决策框架"，回答 design review 当下的问题；05-07 是"纵深 + 外部世界"，回答 12-36 个月时间尺度和组织边界外的问题。先写 01-04 再写 05-07 是为了让叙事链路从"系统 → 组织 → 决策 → 自评 → 不可逆 → 预算 → 外部契约"递进，每章建立在前一章基础上。
- **每章必有 4-5 件可交付产出物**：这是本次发布的核心贡献——把抽象的"架构师水准"落到具体可拿出来开会的工件上（蓝图、RACI、判据矩阵、自评清单、不可逆清单、Day N 状态图、外部契约清单等）。
- **不全书润色**：v1.6 刚做过一轮中文润色，再做是过度工程；本次发布只在新章节坚守同一的"反直觉论断 + 可操作判据 + 反模式 + 不讨论什么 + 接下来"五段结构，与全书风格一致。
- **不动理念三章和知识六章**：这些章节观点已经锐利且有读者反馈验证，不需要修改。

### 这次发布兑现了什么

- 书的标题"架构师之路"——之前的版本主要是 AI SRE 知识，"架构师"层面的产出物是 TODO；本次发布把 7 篇架构成稿写齐

- Capstone 评审包现在有了"架构师视角"的完整工具集（决策矩阵、自评清单、不可逆识别）
