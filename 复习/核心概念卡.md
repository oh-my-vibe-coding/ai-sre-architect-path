## 第一部分 · 引章 & 大模型基础

### 引章：大模型速览

**Q1** · LLM 的一句话本质是什么？
**A** · 一个下一 token 的概率预测器（在海量文本上压缩出的统计引擎）。

**Q2** · 什么是 Context Window？越长越好吗？
**A** · 一次对话能塞的 token 数。**不是越长越好**——长上下文末端服从度下降（Lost in the Middle）。

**Q3** · Temperature=0 和 Temperature=1 的区别？
**A** · 0 最确定，1 发散。SRE 场景多用 0-0.3。

**Q4** · Tool use / function calling 的本质？
**A** · 模型只会**描述**想做什么，实际执行由宿主代码完成。

**Q5** · RAG 是什么？
**A** · Retrieval-Augmented Generation。查询 → embedding → 向量库检索 → 拼进 prompt → 生成。

**Q6** · 4 种必见失败模式？
**A** · 幻觉、截断、注入、漂移。

## 第二部分 · 知识章节

### 第 4 章 · 复合 AI 可靠性数学

**Q** · 每步 95% 正确的 10 步 Agent 端到端正确率约多少？
**A** · ~60%（0.95^10 ≈ 0.599）。

**Q** · 复合系统可靠性工程的两件新工作？
**A** · Step 预算 + Verifier 插入点。

### 第 5 章 · AI 推理服务可靠性

**Q** · LLM 推理服务必须分开定 SLO 的两个指标？
**A** · TTFT（首 token 延迟）和 tokens/s（输出速度）。

**Q** · 容量规划新三角？
**A** · GPU 显存 × 上下文长度 × QPS，三者互相约束。

**Q** · 静默降级（Silent Degradation）是什么？
**A** · 指标全绿但系统数学已经坏了（bf16/fp32 失配、XLA 怪癖、kernel 非确定性）。

### 第 6 章 · AI 自治与上下文约束

**Q** · Lethal Trifecta 三个组成？
**A** · 不受信输入 + 工具访问 + 外泄通道。三者共存 = 攻击可行。

**Q** · Prompt injection 的正确防御层在哪？
**A** · 基础设施层（沙箱 / egress 白名单 / capability-scoped 凭证），不是 prompt 层。

### 第 7 章 · 质量观测与 Data Flywheel

**Q** · 为什么 trace 和 eval 应该是一套系统？
**A** · Traces 就是 eval 数据集，evals 就是生产监控。分成两套会数据断裂、重复投入。

**Q** · 三层 Eval 分别是？
**A** · L1 Assertion（硬规则）/ L2 Judge 模型 / L3 A/B + 用户反馈。

## 第三部分 · 深入专题

### 深入 01 · TTFT 与吞吐

**Q** · Prefill 和 Decode 的瓶颈分别是？
**A** · Prefill 是 **compute-bound**（GPU 算力），Decode 是 **memory-bandwidth-bound**（HBM 带宽）。

**Q** · 为什么 Decode 每生成一个 token 都要读整个模型？
**A** · 每步需要把**全部参数**从 HBM 读到计算单元，还要读所有 KV cache。

**Q** · 70B bf16 模型在 H100 上理论 tokens/s 上限约多少？
**A** · ~24 tokens/s（HBM 3.35 TB/s ÷ 140GB 参数）。

**Q** · TTFT 主要由什么决定？
**A** · 输入 token 数（prefill 时间线性相关于输入长度）。

### 深入 02 · Prompt Caching

**Q** · Prompt caching 缓存的到底是什么？
**A** · **KV 张量**（每层每头的 K 和 V），不是最终回答。

**Q** · 缓存命中的判定条件？
**A** · **Token 级前缀匹配**（不是字符串相等），必须从 token 0 开始连续匹配。

**Q** · Anthropic 缓存读取成本是常规的多少？
**A** · ~0.1×（折扣 90%），写入 1.25×，重用 2 次回本。

**Q** · 为什么不能在 prompt 开头放时间戳？
**A** · 时间戳每次不同 → tokenize 后不同 → 缓存全失效。

### 深入 03 · 模型与工具选型

**Q** · LM Arena 测的是什么，弱点在哪？
**A** · 用户盲测偏好（Elo）。弱点：偏主观对话，不一定反映编码/推理硬实力。

**Q** · SWE-bench Verified 测什么？
**A** · 真实 GitHub issue 修复，是 Agent 编码能力的黄金标准。

**Q** · 选型的 7 步？
**A** · 定义场景 → 硬约束 → 软约束 → 短名单 3 个 → 自己跑 20-50 条 eval → 小流量上线 → 扩量 or 回退。

### 深入 04 · 为什么"你好"消耗数万 token

**Q** · Claude Code 首次"你好"约消耗多少 input token？
**A** · ~14-25k。

**Q** · 这些 token 的主要去向（按占比）？
**A** · System prompt (~37%) + 工具定义 (~49%) + CLAUDE.md 等 (~14%)，用户消息本身仅 2 token。

**Q** · 为什么 LLM API 每次必须重传全部上下文？
**A** · **LLM API 无状态**——模型没有记忆，每次调用必须携带全部上下文。

### 深入 05 · 容量规划

**Q** · 单请求 KV cache 大小公式？
**A** · `2 × num_layers × num_heads × head_dim × bytes_per_element × token数`。

**Q** · GQA 能把 KV cache 缩多少？
**A** · 约 8-16×（Llama 3 70B 用 KV heads=8，相比 64 减 8×）。

**Q** · 训练 vs 推理集群的核心差异？
**A** · 训练是全集群一个故障域、IB 必须；推理是请求级独立、可 scale out。

### 深入 06 · Eval Pipeline

**Q** · Judge 模型需要做什么定期校准？
**A** · 抽样 50-100 条让人工 + judge 都打分，算 **Cohen's κ**，目标 ≥ 0.6。

**Q** · Eval 自己需要的 SLO 有哪些？
**A** · 覆盖率、评估延迟、Judge 对齐度、pipeline uptime、失败率、数据新鲜度。

**Q** · Data Flywheel 的完整循环？
**A** · 生产 trace → 标注失败集 → eval 更新 → prompt/模型变更 → rollout → 新 trace。

### 深入 07 · Prompt Injection 红队

**Q** · 攻击 6 类？
**A** · Direct / Indirect / Tool Poisoning / Exfiltration / Loop Hijacking / History Poisoning。

**Q** · 4 层纵深防御哪层最强？
**A** · **基础设施层**（沙箱 / egress / capability-scoped 凭证）。Prompt 层最弱。

**Q** · Indirect injection 为什么最狡猾？
**A** · 攻击者把 payload 埋在 Agent 会读到的**外部数据**里，输入面是"正常业务数据"无法 pre-filter。

### 深入 08 · 大模型记忆

**Q** · LLM 记忆的三个层次？
**A** · 参数记忆（训练固化）/ in-context 记忆（当前窗口）/ 持久记忆（应用层拼的）。

**Q** · "写数据库作记忆"解决了多少问题？
**A** · ~20%（持久化），剩下 80%（抽取、检索、冲突、污染、元认知、运维）没解决。

**Q** · 持久记忆引入 prompt injection 的什么新风险？
**A** · **注入一次、永久生效**——把单次攻击变成持久威胁。

**Q** · 当前最被推崇的记忆模式？
**A** · **工作笔记模式**（Claude Code 的 NOTES.md）——Agent 自己写、人类可审计、文件系统自带版本。

## 第四部分 · 科学专题

### 科学 01 · Attention

**Q** · Self-attention 的三个角色 Q/K/V 分别表示什么？
**A** · Q=我在找什么（query），K=我是什么（key），V=我的信息（value）。

**Q** · Attention 的计算复杂度？
**A** · **O(N²)**——N 个位置互相算相似度。长 context 成本的物理根因。

**Q** · KV Cache 为什么能节省计算？
**A** · 每生成新 token 时，**历史 token 的 K 和 V 完全不变**，可存下来复用。

**Q** · Attention Sink 是什么？
**A** · 模型在 softmax 里需要一个"出口"倾倒多余注意力，前几个 token 成了默认"垃圾桶"——解释了为什么前置内容受重视。

### 科学 02 · Lost in the Middle

**Q** · 长 context 中间塌陷的 4 个叠加原因？
**A** · 训练分布（重要信息多在首尾）+ 位置编码几何偏差 + softmax 稀释 + attention sink 效应。

**Q** · "宣称 context"和"有效 context"的差距？
**A** · RULER：宣称 ≥32K 的模型**只有一半**能真正用到 32K。Gemini 1.5 1M 实测有效 >128K。

**Q** · 长 context 能替代 RAG 吗？
**A** · **不能**。中间塌陷 + 成本线性涨 + 延迟高。RAG + Rerank 仍是必要的。

### 科学 03 · Quantization

**Q** · Outlier features 是什么，为什么毁掉量化？
**A** · 少数激活维度值极大（±100+），其他维度都在 ±1 以内。int8 只有 256 级，覆盖 outlier 就丢普通值精度。

**Q** · GPTQ 和 AWQ 的核心思想分别是？
**A** · GPTQ 用 Hessian 信息决定哪些权重重要并补偿误差；AWQ 认为"激活大的维度对应权重更重要"，保护这些。

**Q** · 量化静默降级的典型症状？
**A** · **整体 eval 通过但某类任务崩**（数字/结构化输出对量化特别敏感）。必须按任务类型分桶监控。

### 科学 04 · Tokenization

**Q** · 同样信息量，中文在 GPT tokenizer 下的 token 数比 Qwen 多多少？
**A** · ~2-3×（GPT 对中文每字 1.5-2 token，Qwen 每字 0.6-1 token）。

**Q** · LLM 数学差的根因？
**A** · Tokenization 不稳定——"17234" 可能切成 ["17","234"] 也可能 ["1","7","2","3","4"]。数字不是位值表示。

**Q** · Opus 4.7 换 tokenizer 对账单的影响？
**A** · 同样内容 token 数比 Sonnet 4.6 多约 **35%**（1M context 对应 ~555k 词 vs Sonnet 4.6 的 ~750k 词，意味着 1.35× token 体积）。单价相同时实际成本高 ~35%。

## 第五部分 · 共同语言

### 共同语言 01 · 训练生命周期

**Q** · 训练三段是哪三段？
**A** · Pre-training（世界知识）/ Mid-training（聚焦能力提升）/ Post-training（对齐）。

**Q** · SFT 和 RLHF/DPO 的区别？
**A** · SFT 教格式（"被问就答"），RLHF/DPO 教偏好（两个都对的回答里选更好的）。

**Q** · Loss spike 的 SRE 处理流程？
**A** · 暂停 → 检查最近数据批次 → 从最近 checkpoint 恢复 → Skip 可疑数据 → 继续。

**Q** · Model card 至少要看哪 5 项？
**A** · Training data（含 cutoff date）/ Architecture / Recipe / Post-training / Evaluations + regressions。

### 共同语言 02 · Data

**Q** · ML 团队时间主要花在哪？
**A** · 80% 在**数据**（pipeline、清洗、去重、污染检测、混合比例），不是调模型。

**Q** · Pre-training data pipeline 的 7 步？
**A** · Crawl → Clean → Filter → Dedup → Decontaminate → Mix → Tokenize。

**Q** · Contamination 是什么？如何检测？
**A** · 训练数据混入 eval set。检测：N-gram 匹配、canary tokens、private eval。

**Q** · Annealing mix 是什么？
**A** · 训练末期 10-20% steps 用**更高质量、更聚焦**的数据混合（数学、代码比例拉高）。近因效应。

### 共同语言 03 · Research Eval

**Q** · pass@k 的含义？
**A** · 让模型对同一题生成 k 次，至少一次通过的概率。**pass@1 才代表单次可用**。

**Q** · MMLU 为什么区分度下降？
**A** · 饱和（前沿模型 85-90%，接近人类 90%），已无法区分顶级模型。替代是 MMLU-Pro / GPQA。

**Q** · LM Arena Elo 是什么？
**A** · 用户盲测对弈 → Elo 更新。相对分数，**分差比绝对值重要**。差 20 分显著，差 5 分是噪声。

**Q** · Benchmark contamination 怎么识别？
**A** · 某单一指标飙升而 Arena / private eval 不动 + 强项不合理集中 → 大概率刷榜。

### 共同语言 04 · Alignment

**Q** · HHH 三原则？
**A** · Helpful / Honest / Harmless。对齐的三目标。

**Q** · DPO 相比 RLHF 的优势？
**A** · 不训 Reward Model，直接用偏好对比做训练。简单 100×、稳定、效果接近。

**Q** · Reward hacking 是什么？举一个例子？
**A** · Policy 找到 Reward Model 的漏洞获得高分但实际不是好回答。例：RM 偏好长回答 → Policy 学会啰嗦。

**Q** · Jailbreak 和 Prompt Injection 的区别？
**A** · Jailbreak 是"让模型说违规内容"；Prompt Injection 是"让 Agent 做违规动作"。对 SRE 后者更严重。

### 共同语言 05 · 分布式训练

**Q** · FSDP / ZeRO-3 的核心思想？
**A** · 把模型参数、梯度、优化器状态都切片到不同 GPU，forward/backward 时临时 gather。兼得 DP 简单和 TP 省显存。

**Q** · 训练和推理集群的网络要求差异？
**A** · 训练必须 **InfiniBand / NVLink**（每 step 传数百 GB）；推理 Ethernet 足够。

**Q** · Checkpoint 的 SRE 关键指标？
**A** · 写耗时、写失败率、存储容量趋势、最近 checkpoint 年龄、跨区复制延迟。

**Q** · MFU（Model FLOPs Utilization）目标？
**A** · 实际 FLOPs / 峰值 FLOPs，**60-70% 是生产目标**。低于 50% = 效率问题。

## 第六部分 · 学习方法与训练动作

**Q** · B1 动作的核心提问是什么？
**A** · "我错在'不知道'还是'没想到'？"——前者补知识，后者是思维钝化信号。

**Q** · B2 动作的三步？
**A** · 自己写初稿（不许 AI 写）→ AI 挑错（不许重写）→ 自己改。

**Q** · B3 渐进节奏？
**A** · 第 1 月每周 45min → 第 2-3 月每周 90min → 第 4 月+ 每月 3-4 小时。

**Q** · 每月自检的 3 问题？
**A** · (1) 有没有"自己想通本想问 AI 的问题"？(2) B3 有没有完成？(3) B1 方向感有没有提升？
