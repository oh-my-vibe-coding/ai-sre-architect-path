# Unit 4 · Week 1 · Step 预算与端到端正确率

> [← Unit 4 总览](总览.md)  ·  [← 返回目录](../../README.md)

## 本周目标

建立 **"复合系统 reliability ≠ 单模型 SLO"** 的心智模型——能对真实 Agent 系统**算出端到端正确率**的数字。

## 任务清单

### 准备（15 分钟）
- [ ] 选一个**真实的或半真实的 Agent 系统**（不是假想）：
  - 你团队在建的
  - 或一个开源 Agent（Devin、GPT Researcher 等）
  - 或 Claude Code 的一个典型 workflow
- [ ] 能说清楚这 Agent **做什么**、有几个 **step**

### 阅读 · B3 · 45 分钟（无 AI）

**主读**：Chip Huyen · 《Agents》(2025)
  - URL: https://huyenchip.com/2025/01/07/agents.html

**辅读**（15 分钟）：BAIR · 《The Shift from Models to Compound AI Systems》
  - URL: https://bair.berkeley.edu/blog/2024/02/18/compound-ai-systems/

**重点**：
- Chip 对"Agent complexity"的分级
- 为什么"模型变强"不等于"复合系统变可靠"
- 单模型 SLO vs 复合系统 SLO 的区别

### 产出 · B2 · 60-90 分钟

#### Section 1 · Agent 链路拆解

画出你选的 Agent 的**完整 step 图**：

```
[输入] → Step 1: 理解意图
       → Step 2: 检索相关文档
       → Step 3: 生成初步方案
       → Step 4: 用 tool 验证
       → Step 5: 修正
       → Step 6: 输出最终回答
       → [输出]
```

要清晰标出：
- 每步的**模型调用**还是**工具执行**
- 每步的**可能失败方式**（LLM 错答 / tool 报错 / 超时 / ...）

#### Section 2 · 每步正确率估计

**关键**：给每步估一个 p_i（该步正确率）。估的方法：

- 如果有数据（eval 分数 / 历史 log）→ 用数据
- 如果没有 → **保守估计**：
  - 简单分类：0.95
  - 文档检索：0.80-0.90
  - 工具调用：0.90（API 可能失败）
  - 复杂推理：0.70-0.85
  - 开放性生成：0.70-0.80

**最重要**：**标注依据和不确定性**。"我估 0.85，因为 eval 集上是 0.87，但生产 workload 更杂 +/- 5%"。

#### Section 3 · 端到端正确率计算

**独立假设版本**：

```
P(端到端正确) ≈ p1 × p2 × p3 × ... × pN
```

算出你链路的数字。

**非独立修正**：
- 如果 Step N 依赖 Step N-1 的输出，错误会**放大**
- 某些链路有 fallback（错了重试），会**提升**

给出一个区间：**乐观情况 vs 悲观情况**。

#### Section 4 · 直觉检查

- 你算出的端到端数字和**业务的感知成功率**差多少？
- 如果差得多（比如算的 60%，业务感觉 85%）——**为什么**？
  - 可能：业务只在"成功时"用，失败时绕过
  - 可能：多次重试累积成功
  - 可能：你的估算太保守

#### Section 5 · 5 步的算术一下（200 字）

用 Claude 5 步链路做例子：

- 每步 95%：端到端 = 0.95^5 ≈ 77%
- 每步 90%：0.90^5 ≈ 59%
- 每步 85%：0.85^5 ≈ 44%

**关键洞察**：单步小幅退化会让端到端大幅跳水。所以**复合系统的 SLO 必须不同于单模型**。

### AI 挑错

**挑战问题**：
- "我的 p_i 估值有多少是**拍脑袋**、多少是数据支撑？"
- "独立假设成立吗？哪些步骤的失败实际是相关的？"
- "有没有漏掉的 step？"（例：silent 错误 / tool 返回错但 LLM 认为对）

### 预测 · B1 · 每日 5 分钟

本周每次讨论 Agent 系统，先猜：
- "这个链路多少 step？"
- "端到端正确率大概多少？"
- "瓶颈在哪一步？"

## 周末自检

- [ ] 选了**真实系统**（不是纸上虚构）
- [ ] 所有 step 标注了 p_i 和**估值依据**
- [ ] 端到端正确率有数字（不是"大概挺高"）
- [ ] 识别出**和业务感知的差距**并解释

**未达标的表现**：
- 每个 p_i 都是 0.95（没差异化）
- 没做直觉检查
- 独立假设不讨论
